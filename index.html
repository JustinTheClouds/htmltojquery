Htmltojquery

Easily convert any html element into almost any jquery plugin with little or no javascript needed.

Download this project as a .zip file Download this project as a tar.gz file

HTMLToJQuery is a simple plugin to make using all other jquery plugins even easier.

For examples, I will be using the jQueryui plugin, but the HTMLToJQuery plugin works on most other plugins too.

Quick usage

Include the htmltojquery plugin after jquery and after other plugins you will be using.

<script src="http://code.jquery.com/jquery-1.8.3.js"></script>
<script src="http://code.jquery.com/ui/1.9.2/jquery-ui.js"></script>
<script src="javascripts/jquery.htmltojquery.js"></script>

Add the data-plugins attribute to your elements that will be initialized with your favorite plugins.

<div id="accordion" data-plugins="accordion">
	<h3>Section 1</h3>
	...
</div>

Initialize the htmlToJquery plugin

<script>
	$(document).ready(function() {
		$(this).htmlToJquery();
	});
</script>

Define Options

Use data-[pluginname]-[option]="[value]"

<div id="accordion" data-plugins="accordion" data-accordion-collapsible="true">
	<h3>Section 1</h3>
	...
</div>
More information

Events Callbacks

Use data-[pluginname]-[event]="[functionName]"

<div id="accordion" data-plugins="accordion" data-accordion-activate="accordionActivate">
	<h3>Section 1</h3>

</div>

Defaulting Options

Define function setDefaults[Pluginname]Plugin. Then set default options.

function setDefaultsAccordionPlugin(opts) {
	opts.heightStyle = "content";
}

Manipulating Options Before Initialization

Define function preInit[Pluginname]Plugin. Then run code to edit and set options.

function preInitAccordionPlugin(opts) {
	if(x == y) {
		opts.heightStyle = "content";
	}
}

Code for this example

All you need to add is the data-plugins attribute with the plugins you want this element to be initialized with.

<div id="accordion" data-plugins="accordion">
	<h3>Section 1</h3>
	...
</div>

Then just make sure to initialize the htmlToJquery plugin after the DOM is ready.

<script>
	$(document).ready(function() {
		$(this).htmlToJquery();
	});
</script>

Initializing the htmlToJquery plugin on the $(document) will find all elements with the data-plugins attribute. If you would like to target a specific element, you can also use:

$('#targetElement').htmlToJquery();

Targeting a specific element will also include any children elements that also have the data-plugins attribute.

Defining more options for a plugin

Let's say we wanted to set the accordions heightStyle option equal to "content". Typically that would be done during intialization like this:

$( "#accordion" ).accordion({ heightStyle: "content" });

Using the HTMLToJquery plugin, all we need do is add another attribute. data-accordion-height_style="content" 

<div id="accordion" data-plugins="accordion" data-accordion-height_style="content">
	<h3>Section 1</h3>
	...
</div>

Notice we used height_style instead of heightStyle. This is done when camel case is used in the name of an option. The underscore is removed and the following character is capialized automatically.

Simple as adding the function name as the option. data-accordion-activate="accordionActivate"

<div id="accordion" data-accordion-activate="accordionActivate" data-accordion-height_style="content" data-accordion-icons-header="ui-icon-circle-arrow-e" data-accordion-icons-active_header="ui-icon-circle-arrow-s"  data-plugins="accordion">
	<h3>Section 1</h3>
	...
</div>

Then defining the function

function accordionActivate(event, ui, element) {
	alert(ui.newHeader.text() + " is now expanded.");
}

The function will be called with the original arguments from the plugin plus one additional argument. For ex. The jQueryUI accordion plugin has the event activate which passes in the event and ui arguments. There will also be a third argument, element.

Defaulting options

Sometimes you may have some default options you would like set for all elements that will be using a plugin.

This can be achieved by declaring a function named setDefaultsAccordionPlugin. If this plugin is defined, it will be called for every plugin being initialized.

The opts param can be used to set all the default options you like.

function setDefaultsAccordionPlugin(opts) {
	opts.heightStyle = "content";
}

Depending on which plugin you are applying to the element, replace "Accordion" with your plugins name. For ex. "setDefaultsDatepickerPlugin" or "setDefaultsTooltipPlugin".

Setting default options can be overwritten by just declaring the same option as an attribute. So if we did not want heightStyle="content" for one of our elements. We can override it by adding the data-accordion-height_style="auto" attribute to the element.

Manipulating options pre-initialization of a plugin

There may be a time when you need to run some code before a plugin is initialized on an element to effect its options.

This works the same as setting defaults.

Declare the function:

function preInitAccordionPlugin(opts) {
	opts.heightStyle = "content";
}

FAQ

What's the difference between the setDefaultsNamePlugin and the preInitNamePlugin?

The setDefaultsNamePlugin is only executed once, and the the default options are cached.

The preInitNamePlugin is executed for each element that the plugin is found on.

Can you have two plugins on a single element?

Yes! Just separate each plugin name with a space. data-plugins="tooltip draggable"
